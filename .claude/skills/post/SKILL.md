# Post Skill

블로그 포스팅 작성 및 관리 워크플로우

## 역할

Chirpy Jekyll 포맷에 맞는 포스팅을 작성하고, 참조 프로젝트와의 연관관계를 관리한다.

## 트리거

- "post", "포스팅"

## 워크플로우

### 1. 새 포스팅 작성

```
Phase 1: 주제 및 구조 협의
Phase 2: Front Matter 생성
Phase 3: 본문 작성
Phase 4: 참조 프로젝트 등록
Phase 5: 커밋
```

#### Phase 1: 주제 및 구조 협의

사용자와 다음을 확정한다:

- 제목
- 상위 카테고리 선택:
  - **개발 기록**: 프로젝트 경험기, 도구 적응기
  - **생각과 방법론**: 사상, 철학, 설계 원칙, 의사결정
  - **기술 노하우**: 운영 경험 기반 실전 지식
  - **아티클 정리**: 외부 글 요약/리포스팅
- 시리즈명(하위 카테고리): 기존 시리즈 우선 재사용, 신규 생성 가능
- 태그
- 본문 구조 (섹션 목차)

> `categories` 필드는 반드시 2단계 배열 사용: `[상위카테고리, 시리즈명]`

#### Phase 2: Front Matter 생성

```yaml
---
title: "제목"
date: YYYY-MM-DD HH:MM:SS +0900
categories: [상위카테고리, 시리즈명]
tags: [태그1, 태그2]
description: >-
  요약 설명
---
```

- `date`는 작성 시점 기준
- `description`은 2~3문장으로 핵심 내용 요약

#### Phase 3: 본문 작성

- `##`부터 시작 (h1은 title이 대체)
- 코드 블록에 언어 명시
- Mermaid 다이어그램 활용 가능
- 마지막 줄: `*이 글은 Claude의 도움을 받아 작성했습니다.*`

#### Phase 4: 참조 프로젝트 등록

포스팅이 외부 프로젝트를 참조하는 경우, `CLAUDE.md`의 **참조 프로젝트** 테이블에 추가한다.

| 항목 | 설명 |
|------|------|
| 프로젝트 | GitHub 레포 링크 |
| 참조 포스팅 | 발행 후 URL |
| 참조 요소 | 변경 시 포스팅 업데이트가 필요한 구체적 요소 |

#### Phase 5: 커밋

파일명: `_posts/YYYY-MM-DD-slug.md`

### 2. 기존 포스팅 편집

1. `_posts/` 디렉토리에서 대상 포스팅 확인
2. 수정 사항 반영
3. 참조 프로젝트 변경 여부 확인

### 3. 참조 프로젝트 변경 추적

참조 프로젝트에 변경이 발생한 경우:

1. `CLAUDE.md`의 참조 프로젝트 테이블에서 영향받는 포스팅 확인
2. 참조 요소 기준으로 포스팅 업데이트 필요 여부 판단
3. 필요 시 포스팅 내용 수정

## 규칙

- 포스팅은 반드시 Chirpy front matter 형식을 준수한다
- 참조 프로젝트가 있는 포스팅은 반드시 CLAUDE.md에 등록한다
- 카테고리와 태그는 기존 값을 우선 재사용한다
