# Post Skill

블로그 포스팅 작성 및 관리 워크플로우

## 트리거

- "post", "포스팅"

## 사전 조건

- CLAUDE.md에 포스팅 규칙이 정의되어 있어야 한다

## 워크플로우

### 1. 새 포스팅 작성

```
Phase 1: 주제 및 구조 협의
Phase 2: 본문 작성
Phase 3: 비평가 검토 (필수)
Phase 4: 참조 프로젝트 등록
Phase 5: 커밋
```

#### Phase 1: 주제 및 구조 협의

사용자와 제목, 카테고리, 태그, 본문 구조(섹션 목차)를 확정한다.
카테고리와 태그는 기존 값을 우선 재사용한다.

#### Phase 2: 본문 작성

CLAUDE.md의 포스팅 규칙(Front Matter, 작성 규칙, 마크다운 줄바꿈 규칙)을 따른다.

#### Phase 3: 비평가 검토

본문 작성 후 반드시 아래 기준으로 검토한다.
통과하지 못하면 수정 후 재검토한다.

**품질 기준**

| 기준 | 검토 질문 |
|------|-----------|
| 주제 선명도 | 한 문장으로 요약되는가 |
| 분량 적절성 | 불필요하게 길지 않은가 (과한 데이터는 독) |
| 톤과 진정성 | 강의조가 아닌 경험 서사인가 |
| 포트폴리오 임팩트 | 저자의 판단과 성장이 드러나는가 |
| 중복 여부 | 기존 포스팅과 겹치는 내용이 없는가 |

**가독성 기준 (A4 PDF 최적화)**

근거: Google/Microsoft Style Guide, Nielsen Norman Group 웹 가독성 연구,
WCAG CJK 타이포그래피 권장사항 종합.

| 기준 | 검토 질문 |
|------|-----------|
| 문단 단위 | 1문단 = 1아이디어인가 (아이디어가 바뀌면 문단을 나눈다) |
| 문단 길이 | 2-4문장인가 (1문장은 파편화, 6문장 초과는 과밀) |
| 주제문 선행 | 문단 첫 문장이 핵심인가 (독자는 첫 문장 이후 건너뛸 수 있다) |
| 단문 문단 | 1-2문장 문단이 강조·전환·결론에만 쓰였는가 |
| 열거 vs 산문 | 3개 이상 나열은 목록, 인과·맥락은 산문인가 |
| 코드 블록 | 전후에 설명 산문이 있는가 (코드 없이도 글이 읽히는가) |
| 산문 연속 | 소제목 없이 5문단 이상 이어지지 않는가 |

**문단 분리 판단 기준**

연관된 2개 문장을 같은 문단에 둘지 분리할지 판단한다.

- 하나의 요점을 전개 → 같은 문단 유지
- 독립적 관찰이나 결론 → 문단 분리
- 판단 테스트: "첫 문장을 지우면 둘째 문장이 맥락을 잃는가?"
  - 잃는다 → 같은 문단
  - 독립적으로 성립 → 분리 가능

#### Phase 4: 참조 프로젝트 등록

포스팅이 외부 프로젝트를 참조하면 CLAUDE.md 참조 프로젝트 테이블에 등록한다.
참조 프로젝트 섹션이 CLAUDE.md에 없으면 건너뛴다.

#### Phase 5: 커밋

CLAUDE.md 파일 위치 및 네이밍 규칙에 따라 저장 후 커밋한다.

### 2. 기존 포스팅 편집

1. 대상 포스팅 확인
2. 수정 사항 반영
3. 비평가 검토 (Phase 3 적용)
4. 참조 프로젝트 변경 여부 확인

### 3. 참조 프로젝트 변경 추적

1. CLAUDE.md 참조 프로젝트 테이블에서 영향받는 포스팅 확인
2. 참조 요소 기준으로 업데이트 필요 여부 판단
3. 필요 시 포스팅 수정
