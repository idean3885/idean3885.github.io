# Post Skill

블로그 포스팅 작성 및 관리 워크플로우

## 역할

프로젝트의 CLAUDE.md 포스팅 규칙에 맞는 포스팅을 작성하고, 참조 프로젝트와의 연관관계를 관리한다.

## 트리거

- "post", "포스팅"

## 사전 조건

- `CLAUDE.md`에 포스팅 규칙이 정의되어 있어야 한다
- 포스팅 규칙이 없으면 사용자에게 안내 후 중단

## 워크플로우

### 1. 새 포스팅 작성

```
Phase 1: 주제 및 구조 협의
Phase 2: Front Matter 생성
Phase 3: 본문 작성
Phase 4: 참조 프로젝트 등록
Phase 5: 커밋
```

#### Phase 1: 주제 및 구조 협의

사용자와 다음을 확정한다:

- 제목
- 카테고리: CLAUDE.md에 정의된 카테고리 구조를 따름
- 태그: 기존 태그 우선 재사용, 신규 생성 가능
- 본문 구조 (섹션 목차)

#### Phase 2: Front Matter 생성

CLAUDE.md에 정의된 Front Matter 규칙을 따른다.

- 필수 항목이 모두 포함되었는지 검증
- 날짜는 작성 시점 기준

#### Phase 3: 본문 작성

CLAUDE.md의 작성 규칙과 마크다운 줄바꿈 규칙을 따른다.

- 코드 블록에 언어 명시
- 다이어그램 활용 가능

#### Phase 4: 참조 프로젝트 등록

포스팅이 외부 프로젝트를 참조하는 경우, CLAUDE.md에 참조 프로젝트 섹션이 있으면 등록한다.

| 항목 | 설명 |
|------|------|
| 프로젝트 | GitHub 레포 링크 |
| 참조 포스팅 | 발행 후 URL |
| 참조 요소 | 변경 시 포스팅 업데이트가 필요한 구체적 요소 |

> 참조 프로젝트 섹션이 CLAUDE.md에 없으면 이 단계는 건너뛴다.

#### Phase 5: 커밋

CLAUDE.md에 정의된 파일 위치 및 네이밍 규칙에 따라 파일을 저장한다.

### 2. 기존 포스팅 편집

1. 포스팅 디렉토리에서 대상 포스팅 확인
2. 수정 사항 반영
3. 참조 프로젝트 변경 여부 확인

### 3. 참조 프로젝트 변경 추적

참조 프로젝트에 변경이 발생한 경우:

1. CLAUDE.md의 참조 프로젝트 테이블에서 영향받는 포스팅 확인
2. 참조 요소 기준으로 포스팅 업데이트 필요 여부 판단
3. 필요 시 포스팅 내용 수정

## 규칙

- 포스팅은 반드시 CLAUDE.md에 정의된 포맷을 준수한다
- 참조 프로젝트가 있는 포스팅은 CLAUDE.md에 등록한다 (참조 프로젝트 섹션이 있는 경우)
- 카테고리와 태그는 기존 값을 우선 재사용한다
