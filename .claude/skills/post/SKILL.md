# Post Skill

블로그 포스팅 작성 및 관리 워크플로우

## 트리거

- "post", "포스팅"

## 사전 조건

- CLAUDE.md에 포스팅 규칙이 정의되어 있어야 한다

## 워크플로우

### 1. 새 포스팅 작성

```
Phase 1: 주제 및 구조 협의
Phase 2: 본문 작성
Phase 3: 비평가 검토 (필수)
Phase 4: 참조 프로젝트 등록
Phase 5: 커밋
```

#### Phase 1: 주제 및 구조 협의

사용자와 제목, 카테고리, 태그, 본문 구조(섹션 목차)를 확정한다.
카테고리와 태그는 기존 값을 우선 재사용한다.

#### Phase 2: 본문 작성

CLAUDE.md의 포스팅 규칙(Front Matter, 작성 규칙, 마크다운 줄바꿈 규칙)을 따른다.

#### Phase 3: 비평가 검토

본문 작성 후 반드시 아래 기준으로 검토한다.
통과하지 못하면 수정 후 재검토한다.

| 기준 | 검토 질문 |
|------|-----------|
| 주제 선명도 | 한 문장으로 요약되는가 |
| 분량 적절성 | 불필요하게 길지 않은가 (과한 데이터는 독) |
| 톤과 진정성 | 강의조가 아닌 경험 서사인가 |
| 포트폴리오 임팩트 | 저자의 판단과 성장이 드러나는가 |
| 중복 여부 | 기존 포스팅과 겹치는 내용이 없는가 |

#### Phase 4: 참조 프로젝트 등록

포스팅이 외부 프로젝트를 참조하면 CLAUDE.md 참조 프로젝트 테이블에 등록한다.
참조 프로젝트 섹션이 CLAUDE.md에 없으면 건너뛴다.

#### Phase 5: 커밋

CLAUDE.md 파일 위치 및 네이밍 규칙에 따라 저장 후 커밋한다.

### 2. 기존 포스팅 편집

1. 대상 포스팅 확인
2. 수정 사항 반영
3. 비평가 검토 (Phase 3 적용)
4. 참조 프로젝트 변경 여부 확인

### 3. 참조 프로젝트 변경 추적

1. CLAUDE.md 참조 프로젝트 테이블에서 영향받는 포스팅 확인
2. 참조 요소 기준으로 업데이트 필요 여부 판단
3. 필요 시 포스팅 수정
