# Spec Skill

명세(설계) 워크플로우

## 역할

요구사항을 분석하고 마크다운 명세 문서를 작성한다.

## 프로젝트 프로필 연동

`.claude/project-profile.md` 파일이 존재하면:
1. 프로필의 **`/spec 컨텍스트`** 섹션을 읽는다
2. 해당 컨텍스트에 정의된 설계 대상, 명세 항목, 다이어그램 방식을 따른다
3. 프로필이 없으면 아래 기본 워크플로우대로 동작한다

## 워크플로우

1. **프로필 확인**: `.claude/project-profile.md`를 읽어 프로젝트 특수성을 파악한다.
2. **요구사항 파악**: 사용자에게 질문하여 구현 대상을 명확히 한다.
3. **기존 문서 확인**: 프로젝트 내 관련 README, docs/ 문서를 읽어 컨텍스트를 파악한다.
4. **설계 문서 작성**:
   - 프로필이 있으면: 프로필의 `/spec 컨텍스트`에 정의된 항목 기준으로 작성
   - 프로필이 없으면 (기본값):
     - 아키텍처 설계 (Mermaid 다이어그램)
     - 인터페이스 정의 (입력/출력, API 스펙)
     - 구현 범위 및 제약사항
5. **사용자 검토**: 문서를 제시하고 승인을 받는다.
6. **문서 저장**: 승인 시 `docs/` 또는 해당 모듈 README에 저장한다.

## 산출물

- 설계 문서 (Markdown)
- 다이어그램 (Mermaid 임베딩, 상세 시 PlantUML + SVG)
- 프로필에 정의된 추가 산출물 (있는 경우)

## 규칙

- 코드 작성 금지 (설계만 수행)
- 사용자 승인 전까지 다음 단계로 진행하지 않는다
- CLAUDE.md의 다이어그램 규칙을 따른다 (Mermaid 기본, 상세 시 PlantUML)
- 프로필이 있으면 프로필의 제약사항도 함께 준수한다
