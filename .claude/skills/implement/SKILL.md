# Implement Skill

구현 워크플로우

## 역할

설계 문서를 기반으로 산출물을 구현한다.

## 프로젝트 프로필 연동

`.claude/project-profile.md` 파일이 존재하면:
1. 프로필의 **`/implement 컨텍스트`** 섹션을 읽는다
2. 해당 컨텍스트에 정의된 구현 대상, 구현 순서, 검증 기준을 따른다
3. 프로필이 없으면 아래 기본 워크플로우대로 동작한다

## 전제조건

- 설계 문서가 존재해야 한다 (없으면 사용자에게 `/spec` 실행을 안내)

## 워크플로우

1. **프로필 확인**: `.claude/project-profile.md`를 읽어 프로젝트 특수성을 파악한다.
2. **설계 문서 확인**: 관련 설계 문서(docs/, README)를 읽어 구현 범위를 파악한다.
3. **구현 범위 확정**: 사용자에게 확인하여 이번에 구현할 범위를 확정한다.
4. **구현 수행**:
   - 프로필이 있으면: 프로필의 `/implement 컨텍스트`에 정의된 구현 대상과 순서를 따른다
   - 프로필이 없으면 (기본값): 설계 문서 기준으로 코드를 작성한다
5. **검증**:
   - 프로필이 있으면: 프로필의 검증 기준을 적용한다
   - 프로필이 없으면 (기본값):
     - 빌드 확인 (프로젝트에 빌드 도구가 있으면 실행)
     - 테스트 실행 (테스트가 있으면 실행)
6. **문서-산출물 동기화**: 구현으로 인해 문서 수정이 필요하면 변경사항을 보고한다.

## 규칙

- 설계 문서 기준으로만 구현한다
- 구현 중 설계 변경이 필요하면 사용자에게 먼저 보고한다 (임의 변경 금지)
- 커밋/푸시는 수행하지 않는다 (사용자가 `/commit`으로 별도 진행)
- 프로필이 있으면 프로필의 제약사항도 함께 준수한다
