# ADR-0004: 카테고리 뎁스 2를 다중 글 둥지로 한정

## Status

Accepted (2026-05-20)

## Context

ADR-0003 은 시리즈 운영을 폐지하면서 `categories: [상위카테고리, 하위카테고리]` 뎁스 2 구조는 유지하기로 했다. 당시 "카테고리 뎁스 1로 단순화" 안은 사이드바 라벨 독립 노출과 카테고리 페이지 자동 인덱스를 잃는다는 이유로 기각됐다.

ADR-0003 의 시리즈 압축이 끝난 시점에서 부작용이 드러났다. 연속 서사 시리즈를 신규 1편으로 응축하자, 그 자리의 하위 카테고리가 글 1편만 담은 폴더로 남았다.

운영 6개월, 23편 기준 하위 카테고리는 12개인데 그중 7개가 1편짜리다.

- 트립 매니저(1), 인증서 자동화 구축기(1), Slack to Notion(1), 블로그(1), Python(1), small-peaces(1), Kubernetes(1)

1편짜리 하위 카테고리는 분류 기능을 하지 못한다. 글 하나에 폴더 껍데기를 씌운 것이라 사이드바 펼침 메뉴만 길어진다. 개발 기록 상위 카테고리는 하위 6개 중 절반이 1편짜리라 큐레이션이 아니라 산만함으로 보였다.

반면 2편 이상 모인 하위 카테고리(개발 사전 5, 실무 노하우 3, 미터링 시스템 구축 3, AI 적응기 2, AI 교차 검증 2)는 사이드바에서 글 누적이 보이는 실제 둥지다. 이력서/포트폴리오 관점에서 "개발 사전 5편"이 한 라벨로 보이는 효과는 유지할 가치가 있다.

뎁스 1 글(`categories: [생각과 방법론]`)이 이미 존재하고 Chirpy 가 정상 렌더하므로, 같은 상위 카테고리 안에서 뎁스 1 글과 뎁스 2 글이 섞여도 문제가 없다.

## Decision

1. **하위 카테고리는 2편 이상 모인 둥지에만 부여한다.** 1편짜리 주제는 상위 카테고리만 부여한다(뎁스 1).
2. **신규 글은 기본적으로 뎁스 1로 작성한다.** 같은 하위 주제의 2편째가 생기는 시점에 하위 카테고리를 신설하고, 1편째도 같은 하위 카테고리로 함께 옮긴다.
3. **예외: Kubernetes 는 1편이지만 하위 카테고리를 유지한다.** ADR-0002 가 학습 글 누적을 의도하고 신설한 선행 라벨이므로 2편 기준의 예외로 둔다. 다른 선행 라벨을 새로 만들 때도 ADR 로 의도를 먼저 기록해야 예외로 인정된다.
4. **이번 정리로 1편짜리 하위 카테고리 6개를 뎁스 1로 승격한다.** 트립 매니저, 인증서 자동화 구축기, Slack to Notion, 블로그, Python, small-peaces.

front matter 형식 변경은 없다. `categories` 배열의 원소 개수만 1 또는 2로 달라진다.

## Consequences

긍정:

- 사이드바 펼침 메뉴가 실제로 글이 모인 분류만 노출한다. 1편짜리 껍데기가 사라진다.
- 신규 글이 뎁스 1 기본이라 작성 시 하위 카테고리를 억지로 만들려는 회귀 패턴이 줄어든다.
- 2편 이상 둥지(개발 사전, 실무 노하우 등)의 사이드바 가시성은 그대로 보존된다. ADR-0002 의 강점 가시화 의도와 정렬된다.
- 포스트 URL 은 슬러그 기반이라 카테고리 변경이 URL 을 바꾸지 않는다. 외부 인용 깨짐과 GSC 색인 손실이 없다.

부정:

- 신규 하위 주제는 2편째에 도달하기 전까지 상위 카테고리에 평평하게 쌓인다. 1편 단계에서는 사이드바 라벨로 구분되지 않는다.
- Kubernetes 예외가 "2편 이상" 기준과 어긋나, 규칙에 단서가 붙는다.
- 하위 카테고리를 새로 신설할 때 1편째 글의 `categories` 도 함께 고쳐야 한다(수작업 1건).

## Alternatives Considered

| 대안 | 기각 사유 |
|------|----------|
| 전부 뎁스 1로 평탄화 | 개발 사전(5)·실무 노하우(3) 같은 실제 누적 둥지의 사이드바 라벨이 사라진다. ADR-0002 의 강점 가시화 의도 위배 |
| 현행 유지 (ADR-0003 그대로) | 1편짜리 하위 카테고리 7개가 사이드바를 산만하게 만드는 문제가 그대로 남는다 |
| Kubernetes 도 함께 승격 | ADR-0002 가 명시한 선행 라벨 의도(학습 글 누적 둥지)와 충돌 |
| 2편이 아니라 3편을 하위 카테고리 기준으로 | AI 적응기·AI 교차 검증(각 2편)이 둥지에서 빠진다. 2편이면 이미 "같은 주제 누적"이 성립하므로 기준을 낮게 잡음 |

## ADR-0003 과의 관계

ADR-0003 은 "카테고리 뎁스 2 유지"를 결정했다. 이 ADR 은 그 유지 *범위*를 한정한다. 뎁스 2 메커니즘 자체는 그대로 살아 있고, 다중 글 둥지 5개와 Kubernetes 가 계속 사용한다. 폐지가 아니라 적용 기준을 추가하는 결정이므로 ADR-0003 은 Superseded 가 아니다.

ADR-0003 이 기각했던 "카테고리 뎁스 1로 단순화"는 *전면* 평탄화였다. 이 ADR 은 전면 평탄화가 아니라, 글이 1편뿐인 자리에 한해 뎁스 1 을 적용한다. 두 결정은 충돌하지 않는다.
