# ADR-0003: 시리즈 운영 폐지 + 카테고리 뎁스 2 유지

## Status

Accepted (2026-05-17)

## Context

블로그 초기에 `categories: [상위카테고리, 시리즈명]` 구조로 연속 서사 글을 묶었다. 적응기, 미터링 시스템 구축, 인증서 자동화 같은 시리즈가 이 방식으로 운영됐다.

운영 6개월 시점에서 단점이 드러났다.

- 시리즈에 묶인 글은 "다음 편에서 다룰 예정" 같은 미루기 구조를 만든다. 독자가 1편을 끝까지 읽지 않을 가능성이 높은데, 미루기는 그 가능성을 더 줄인다.
- 한 시리즈가 N편으로 늘어나면 각 편의 인사이트 밀도가 떨어진다.
- 이력서/포트폴리오 관점에서는 핀포인트 1편이 핵심을 더 잘 전달한다.
- CLAUDE.md "현재 시리즈명" 표가 시리즈 운영을 전제로 남아 있어, 신규 글 작성 시 시리즈에 묶으려는 회귀 패턴이 반복됐다.

이미 memory `feedback-single-post-complete` 에서 시리즈 분할 금지 정책이 명시됐으나 ADR 단위로 결정 사유가 기록되지 않아 회귀 위험이 남아 있었다. ADR-0001 의 회귀 방지 목적과 정렬하려면 ADR로 결정 사유를 기록할 필요가 있었다.

## Decision

1. **시리즈 운영을 공식 폐지한다.** "시리즈"라는 단어와 운영 개념을 CLAUDE.md 에서 제거.
2. **카테고리 뎁스 2 구조는 유지한다.** `categories: [상위카테고리, 하위카테고리]` 형식. 사이드바 펼침 메뉴 노출과 카테고리 페이지 자동 인덱스가 이 구조에 의존한다.
3. **2번째 자리는 "하위 카테고리"로 단어 통일한다.** 기존 "시리즈명" 자리에 들어가던 라벨이 그대로 하위 카테고리 라벨이 된다. 신규 글은 1편이 단독 완성형이라는 전제로 작성된다.
4. **연속 서사 기존 시리즈는 시리즈당 신규 1편으로 응축한다.** N편의 핵심 인사이트만 추출한 단독 완성형 1편을 신규 슬러그로 작성하고, 기존 N편은 삭제한다. CLAUDE.md "참조 프로젝트" 표는 신규 URL로 동시 갱신한다.
5. **압축은 한 세션당 1시리즈만 처리한다.** 41편 일괄 정리는 비가역 위험이 크고, 외부 인용 깨짐과 GSC 색인 손실이 시리즈별로 분산되어야 영향 범위가 닫힌다.

front matter 형식 변경 없음. 기존 글의 `categories: [개발 기록, AI 적응기]` 같은 형태가 그대로 유효하며, 단어 의미만 "시리즈" → "하위 카테고리"로 재정의된다.

## Consequences

긍정:

- 신규 글은 1편 단독 완성형 정책이 ADR로 기록되어 회귀가 막힌다.
- 카테고리 뎁스 2 사이드바 노출이 보존된다.
- 압축으로 핀포인트 인사이트가 모이고 이력서/포트폴리오 자산이 누적된다.
- 한 세션당 1시리즈 정책으로 비가역 위험이 분산된다.

부정:

- 시리즈별 압축 작업이 7건 누적된다. 시리즈당 1~2시간 (미터링은 더 길다).
- 외부 인용 URL이 깨지는 시점에 CLAUDE.md "참조 프로젝트" 표 갱신을 누락하면 정합성이 깨진다.
- 압축 1편은 N편의 합보다 훨씬 짧다. 일부 디테일이 사라진다 (git history에는 남는다).

## Alternatives Considered

| 대안 | 기각 사유 |
|------|----------|
| 시리즈 운영 유지 | memory `feedback-single-post-complete` 정책과 충돌. 회귀 위험 그대로 |
| 시리즈는 유지하되 N편 인덱스 페이지 신설 | 미루기 구조와 인사이트 분산 문제는 해결되지 않음 |
| 41편 일괄 정리 한 PR | 비가역 위험 + 외부 인용 동시 깨짐 + GSC 색인 동시 손실 |
| 카테고리 뎁스 1로 단순화 | 사이드바 노출 + 카테고리 페이지 인덱스 손실. ADR-0002 의 강점 가시화 의도 위배 |
| 시리즈마다 베이스 글 1편을 그대로 두고 나머지만 삭제 | 베이스 글이 다른 글들의 인사이트를 흡수하지 못해 단독 완성형이 되지 않음 |

## ADR-0002 와의 관계

ADR-0002 는 "Kubernetes를 별도 시리즈로 노출"이라는 표현을 썼다. ADR-0003 이후 그 자리의 "시리즈"는 "하위 카테고리"로 읽는다. ADR-0002 의 의도(사이드바 라벨 독립 노출, 카테고리 페이지 자동 인덱스)는 변경 없이 유효하다. ADR-0002 본문은 영구 기록 원칙에 따라 그대로 둔다.
